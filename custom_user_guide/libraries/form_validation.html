Form Validation

This is subclass of the original class with more validation rules.

Rule Reference

The following is a list of all the native rules that are available to use:

Rule Parameter Description Example
check_email_list No Returns FALSE if there is an error in a list of emails separated by comma.  
check_phone No Returns FALSE if the form element doesn't contain a valid phone number (it only validates that it contains only numbers and slashes)  
not_matches Yes Returns FALSE if the form element matches the field. It's the opposite of the "matches" rule of the original Codeigniter not_matches[form_item]
check_captcha No Returns FALSE if the Recaptcha field is incorrect.  
valid_url No Returns FALSE if the form element doesn't contain a valid URL.  

Note: These rules can also be called as discrete functions. For example:

$this->form_validation->check_phone($string);
